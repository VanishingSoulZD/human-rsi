# Human RSI系统架构文档（工程实现版 v1）

---

# 0. 系统定义（工程视角）

Human RSI（Human Recursive Self-Improvement）在工程上可以定义为：

> 一个以“人类能力增长”为优化目标的闭环系统，其核心是持续生成、执行、评估并优化“最优成长路径”。

系统不是：

- 学习平台
- Agent框架
- 教练工具

而是：

> **一个“能力优化引擎 + Agent执行系统 + 反馈学习系统”的统一体**

---

# 1. 总体架构（五层系统）

```text id="arch_overview"
┌──────────────────────────────┐
│ L5 Memory System             │ ← 长期状态存储
├──────────────────────────────┤
│ L4 Feedback & Evaluation     │ ← 成长评估 / Reward
├──────────────────────────────┤
│ L3 Task & Agent Execution    │ ← 行动执行层
├──────────────────────────────┤
│ L2 ZPD Engine                │ ← 核心调度层
├──────────────────────────────┤
│ L1 Capability Model          │ ← 能力建模层
└──────────────────────────────┘
```

---

# 2. L1：能力建模层（Capability Model）

## 2.1 作用

定义：

> 用户“当前状态是什么”

---

## 2.2 输入

- 用户行为日志
- 历史任务结果
- 测试/评测结果
- Agent执行记录

---

## 2.3 输出

```json
{
  "skills": {
    "python": 0.85,
    "llm_agents": 0.72,
    "system_design": 0.4
  },
  "cognitive_profile": {
    "abstraction_level": 0.6,
    "execution_speed": 0.7
  },
  "bottlenecks": ["task decomposition", "long-term planning"]
}
```

---

## 2.4 内部机制

- skill embedding
- graph-based skill model
- LLM-based evaluator
- historical trajectory fitting

---

## 2.5 输出去向

→ L2 ZPD Engine

---

# 3. L2：ZPD生成层（核心引擎）

## 3.1 定义

Lev Vygotsky

> 生成“刚好超出当前能力但可学习”的任务空间

---

## 3.2 输入

- Capability Model
- 用户目标（可选）
- 历史任务表现
- 当前上下文

---

## 3.3 输出

```json
{
  "candidate_tasks": [
    {
      "task": "Build a simple agent with memory",
      "difficulty": 0.78,
      "expected_growth": 0.12
    },
    {
      "task": "Implement reflection loop for agent",
      "difficulty": 0.82,
      "expected_growth": 0.15
    }
  ]
}
```

---

## 3.4 核心算法思想

```text id="zpd_core"
ZPD(task) = argmax GrowthGain(task)

subject to:
    Difficulty ≈ Capability + ε
    Motivation > threshold
    Feasibility = true
```

---

## 3.5 内部模块

- difficulty estimator
- skill gap analyzer
- curriculum generator
- constraint checker

---

## 3.6 输出去向

→ L3 Task & Agent Layer

---

# 4. L3：任务调度 + Agent执行层

## 4.1 组成

这一层分两个子模块：

### 4.1.1 Task Scheduler（任务调度器）

### 输入：

- ZPD任务集合
- 用户时间/资源
- 历史失败记录

### 输出：

- 今日任务列表
- 执行顺序

---

### 4.1.2 Agent Execution System

Agent Loop

### 输入：

- 单个任务

### 输出：

- 完成结果
- 执行轨迹

---

### 内部流程：

```text id="agent_loop"
Plan → Act → Observe → Reflect → Retry
```

---

### 工具能力：

- code execution
- search
- reasoning
- tool use
- reflection

---

### 输出去向：

→ L4 Feedback System

---

# 5. L4：反馈与评估层（Reward / Evaluation）

## 5.1 作用

判断：

> 用户是否真的“变强了”

---

## 5.2 输入

- Agent执行结果
- 用户行为
- 任务完成情况
- 外部指标（考试/项目/面试）

---

## 5.3 输出

```json
{
  "task_success": true,
  "quality_score": 0.78,
  "skill_delta": {
    "agent_design": +0.05
  }
}
```

---

## 5.4 Reward Model

Reward Model

核心问题：

> 什么算“成长”？

不是：

- 完成任务

而是：

- 能力提升

---

## 5.5 评估方式

- LLM-as-a-judge
- rubric scoring
- pairwise comparison
- trajectory evaluation

---

## 5.6 输出去向

→ L5 Memory + L1 Capability Update

---

# 10. 系统本质公式（统一表达）

```text id="rsi_formula"
maximize  Growth(Capability)

subject to:
    Task ∈ ZPD(Capability)
    Action = Agent(Task)
    Reward = f(Capability Delta)
```

---

# 11. 与传统Agent系统的差异

| 模块     | 传统Agent        | Human RSI                   |
| -------- | ---------------- | --------------------------- |
| 目标     | 完成任务         | 人的能力增长                |
| 状态     | 外部任务         | 内部能力结构                |
| 核心调度 | Prompt / Planner | ZPD Engine                  |
| 反馈     | success/failure  | capability delta            |
| 记忆     | short-term       | long-term growth trajectory |
