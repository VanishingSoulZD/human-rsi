# Human RSI理论体系文档（V1）

---

# 1. 总体目标：Human RSI是什么系统？

Human RSI（Human Recursive Self-Improvement）可以统一定义为：

> 一个持续优化“人类能力增长路径”的递归系统。

它不是：

- 工具集合
- 学习平台
- Agent框架

而是一个闭环系统：

```text id="rsi_root"
Human State(t)
   ↓
Growth Policy
   ↓
Action Selection
   ↓
Execution (Agent)
   ↓
Feedback
   ↓
Memory Update
   ↓
State(t+1)
```

---

# 2. 系统核心分层结构（最重要）

整个 Human RSI 可以分为 5 层：

---

## L0：环境层（Environment）

定义：

> 用户真实世界中的任务与约束

例如：

- 工作任务
- 学习任务
- 项目
- 时间压力
- 情绪状态

这一层决定：

> 系统“优化什么现实”

---

## L1：能力模型（Capability Model）

### 定义

Capability Model

> 对“人当前能力结构”的结构化表示

---

### 内容包括：

- 技能图谱（Skill Graph）
- 熟练度（Proficiency）
- 知识覆盖
- 执行能力
- 思维能力
- 瓶颈识别

---

### 作用

回答一个问题：

> “这个人现在到底能做什么？”

---

### 输出形式：

```text id="cap_model"
Skill:
  Python: 80
  LLM: 70
  RL: 40
  Product Thinking: 30
```

---

## L2：ZPD成长调度层（核心引擎）

### 定义

Lev Vygotsky

> 近侧发展区（Zone of Proximal Development）

---

### 在 Human RSI 中的作用：

> 生成“刚好最优难度”的下一步任务

---

### 核心问题：

- 什么任务太简单？
- 什么任务太难？
- 什么任务刚好能带来最大成长？

---

### 数学化表达：

```text id="zpd_formal"
Task Difficulty ≈ Capability + ε
```

---

### 本质：

ZPD = Growth Optimization Constraint

---

## L3：策略层（Growth Policy / OODA）

### 包含：

OODA Loop

OODA：

- Observe（观察）
- Orient（分析）
- Decide（决策）
- Act（执行）

---

### 在 Human RSI 中作用：

> 把“能力状态”转换为“行动策略”

---

### 关系：

```text id="ooda_layer"
Capability → OODA → Action Plan
```

---

### 作用：

- 决定下一步做什么
- 决定优先级
- 决定资源分配

---

## L4：执行层（Agent Loop）

### 定义

Agent Loop

> 将任务执行 + 反思 + 修正构成循环系统

---

### 典型结构：

```text id="agent_loop"
Plan → Act → Observe → Reflect → Update → Repeat
```

---

### 关键机制：

- ReAct
- Reflexion
- Voyager
- Tool use
- Self-correction

---

### 功能：

- 执行任务
- 自动拆解任务
- 失败恢复
- 经验积累

---

## L5：记忆系统（Memory System）

### 定义

Memory System

> 存储“经验 + 状态 + 轨迹”的长期系统

---

### 内容包括：

- 行为记录
- 成功/失败案例
- 技能增长轨迹
- 用户偏好
- 历史任务路径

---

### 作用：

> 让系统“知道你是谁，以及你如何变化”

---

# 3. 核心闭环结构（Human RSI主循环）

整合所有层之后，系统是一个闭环：

```text id="rsi_loop_full"
        L5 Memory
             ↓
        L1 Capability Model
             ↓
        L2 ZPD Engine
             ↓
        L3 OODA Policy
             ↓
        L4 Agent Execution
             ↓
        Feedback / Evaluation
             ↓
        Update Memory
             ↓
        Update Capability
             ↓
        Repeat
```

---

# 4. 关键关系（不是并列，而是依赖结构）

## 4.1 Capability → ZPD

- Capability定义“你现在在哪”
- ZPD定义“你下一步去哪”

---

## 4.2 ZPD → OODA

- ZPD决定“候选任务集合”
- OODA选择“最优路径”

---

## 4.3 OODA → Agent

- OODA决定策略
- Agent执行策略

---

## 4.4 Agent → Memory

- 所有执行结果进入记忆
- 用于未来优化

---

## 4.5 Memory → Capability

- 通过历史轨迹更新能力模型
- 形成“成长曲线”

---

# 5. 系统本质（统一视角）

Human RSI 本质是一个：

> **带约束的递归优化系统**

可以抽象为：

```text id="rsi_math"
maximize  Growth(t+1) - Growth(t)
subject to:
    Action ∈ Agent Space
    Difficulty ∈ ZPD(Capability)
    Memory provides constraints
```

---

# 6. 三大核心变量（系统最重要部分）

整个系统只有三个真正核心变量：

---

## 6.1 Capability（状态）

> 你是谁

---

## 6.2 ZPD（约束）

> 你应该做什么难度的任务

---

## 6.3 Reward / Feedback（信号）

Reward Model

> 你是否真的变强了

---

# 7. 三者关系（核心思想）

```text id="core_relation"
Capability → 定义状态
ZPD → 定义下一步边界
Reward → 判断是否有效成长
```

---

# 8. Human RSI vs 传统AI系统差异

| 维度     | 传统 Agent   | Human RSI         |
| -------- | ------------ | ----------------- |
| 优化目标 | 完成任务     | 人的成长          |
| 状态     | 外部任务     | 人的能力          |
| 反馈     | task success | capability growth |
| 时间尺度 | 短期         | 长期              |
| 核心变量 | action       | ZPD + Capability  |

# 9. 系统关键突破点（你真正要做的）

整个 Human RSI 系统的难点只有三个：

---

## 9.1 Capability Model 是否真实

- 是否能准确建模“人”

---

## 9.2 ZPD 是否可计算

- 是否能找到“刚好合适的任务”

---

## 9.3 Feedback 是否可信

- 是否真的测量“成长”，而不是“完成任务”
